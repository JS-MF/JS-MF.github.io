Concepts

The mains concepts developed in JSMF framework.

The Modelling Continuum: Flexibility and Model Checking

Flexibility regarding the situation to be modeled in the key concern of JSMF. However, withtout control flexibilty is just plain data than cannot be manipulated by tools. As a result, model checking facility is also an help to balance between flexibility and computer-based model manipulation.

Model to Model Transformation

Model transformation is one core aspect of model manipulation. Operations/functions that can be made on model can be systematically addressed with model transformations

Model Management

Model management (or megamodel, macromodel) is the key to ensure a coherent and efficient modelling landscape. This is of particular interest when dealing with a multi-perspective, multi-stakeholder modelling situation.

Model Emergence and Data-Driven Modelling

By essence in the MDE (meta)models are structured. However they can come from data and a structure could appear from the analysis of such data. JSMF aims at supporting situation where models are not necessarily the first entities.

History

Rational and timeline.

  • 2012-2014

    Early Discussion about Simplifying Modelling Framework for Web App

    We experience since 2008 the use of traditional Model-Driven Engineering (MDE) tools in Eclipse and found some limitation to our first topic of interest: Web User Interfaces generation. Later on we applied MDE tools to diverses topics which require more flexibility.

  • July 2014

    First Experiments based on JavaScript

    After a meeting with Jean Bezivin we learnt that current Eclipse Modelling Framework development was tightly coupled with Eclipse plateform need. Thus it could not evolve in direction we would like to use modelling tools: more flexibly (e.g., flexible conformance relation).

  • 2015-2016

    Development of our own Modelling Framework

    We developed a first version of a modelling framework following the JavaScript world philosophy.

A Flexible Modelling Framework

https://github.com/JS-MF/jsmf-core

The JavaScript Modelling Framework (JSMF) has been designed for providing a flexible modelling environment that could support the requirement of natural modelling [1]. It is a JavaScript embedded DSL inspired by the Eclipse Modelling Framework (EMF) in its basic functions but that rely on JavaScript dynamic typing and on a relative independence between a metamodel and a model.

  • [1] Z Zarwin, M Bjekovic, JM Favre, JS Sottet, HA Proper, Natural Modelling, Journal Of Object Technologies, 2014

JavaScript Model Transformation

Transforming models with JavaScript

JSTL is a model to model transformation engine that handle JSMF models. JSTL simplifies the transformation writing while leaving some responsibilities to the transformation developer following JavaScript way of thinking (e.g., the developer has to check for concurrent rules).

In JSTL one can define a model transformation using two patterns: in/out. The in pattern is a function that matches input model elements. The out pattern can generate multiple modelling elements.

Managing Heterogeneous Models and MDE artifacts

Model management is key when dealing with a multi-perspective, multi-stakeholder system. It encompasses all the tools necessary for ensuring consistency, being able to classify, relating the different modelling artifacts (i.e., model, metamodels, transformation, modelling elements, etc.).

Model Checking and Model Navigation

Model manipulation fundamentals.

Model Checking is one important think to address when dealing with flexible model: without control dealing with flexibility is like using plain data. In order to get all the power of the MDE approach it is sometimes required to ensure that the model has some property. The first citizen property is the conformance.

A second important point is the navigation/query of model called JSMF-Magellan. In order to facilitate the model handling we provide an API that permit quick model traversing (e.g., following specific relations, properties) or filtering. Magellan can be used as a basic component for model transformation or for higher abstraction API.

Model Discovery

Bridging Data and Models together.

Model Discovery is a versatile term encompassing emergence of (meta)model from model, from data (e.g., using clustering), discovering a structure from a given metamodel pattern (i.e., classifying data), etc.
